# Google Play Store Data Analysis (EDA Project)

# Dataset link - https://raw.githubusercontent.com/krishnaik06/Playstore-Dataset/main/googleplaystore.csv

## Project Overview
This project focuses on performing Exploratory Data Analysis (EDA) on the Google Play Store dataset. The goal is to understand trends in apps, categories, ratings, installs, and other key features to extract meaningful insights about the Android app market.

---

## Objectives
- Clean and preprocess raw Google Play Store data
- Handle missing values and inconsistent formats
- Perform exploratory data analysis (EDA)
- Identify trends in app categories, ratings, and installs
- Extract actionable insights from the dataset

---

## Dataset Information
The dataset contains information about Android apps on the Google Play Store, including:

- App name
- Category
- Rating
- Reviews
- Size
- Installs
- Type (Free/Paid)
- Price
- Content rating
- Genres

---

## Data Cleaning Steps
- Removed duplicate records
- Handled missing values
- Converted data types (e.g., Installs, Price)
- Cleaned textual inconsistencies (e.g., size, installs format)
- Standardized categorical values

---

## Exploratory Data Analysis
Key analysis performed:
- Top app categories by number of apps
- Distribution of ratings
- Most installed app categories
- Free vs Paid app comparison
- Relationship between reviews, ratings, and installs

---

## Key Insights
- Most apps are concentrated in a few major categories like Tools, Family, and Games
- Majority of apps are free
- Higher installs generally correlate with higher reviews
- Ratings are mostly skewed towards higher values (4.0+)
